The core question

Which strike delivers the strongest premium for the risk taken right now?

“Given today’s chain and the phase I am in, which strike offers the best risk adjusted premium per week per dollar of provisioned capital?”

That is the product story in one line. Everything else on this page exists to show how WISE makes that answer rigorous, phase aware, and auditable without becoming a black box.

A phase aware decision system

Philosophy

Five principles that shape every decision.

These are the ideas a first time viewer needs to understand first. The remaining detail shows up later in workflow, scoring, and state.

01

The edge comes from variance risk premium.

WISE starts from a structural idea: implied volatility tends to overstate realized volatility. That is why premium selling can be attractive on both legs of the wheel.

02

Score the phase you are actually in.

Put phase uses the live put chain. Call phase uses the live call chain. WISE does not project future chain economics to make the current decision look cleaner than it is.

03

Assignment is a transition, not a model failure.

When a short put becomes long MES, the engine switches phase and records cost basis. That keeps the wheel unified instead of pretending the put leg is the whole strategy.

04

Rank for premium velocity, then penalize fragility.

The model looks for edge per week per dollar of provisioned capital, then discounts candidates for impairment, proximity, floor, and recovery burden.

05

Every output should be explainable.

There is one config voice, one canonical pipeline, and one surface that shows the reasoning. Auditability is designed in rather than added later.

→

The result is a clearer wheel workflow.

These principles later show up as state separation, workflow gating, and phase specific scoring. The logic stays consistent from the math to the dashboard.

Two engines, one strategy

WISE makes one decision in two distinct phases.

Phase one

Score live put candidates

WISE ranks the real put chain for edge, liquidity, regime fit, and survivability.

  • Shallow ITM can remain eligible
  • Support and capital checks still apply
ASSIGNMENT
Transition

Record cost basis and switch phase

Assignment becomes long MES at the put strike, less collected put premium. The engine changes context instead of treating the event as a mistake.

  • Cost basis is stored automatically
  • Cycle premiums remain attached to the trade
RECOVERY
Phase two

Score live covered calls

The call engine scores extrinsic value only, while keeping carry, dead zone status, and full exit P&L visible.

  • Select the best survivable call candidate
  • Show carry and full cycle economics separately
The point is not to avoid every assignment. The point is to make the next phase legible and score it honestly.

The engine

A formal scoring model. Supportive proof, not the opening act.

The math is there to prove the recommendation is disciplined, not to ask a new viewer to decode equations first. Read the plain language framing, then use the formulas as supporting evidence.

Put score · v9.3

Plain language: WISE rewards premium that looks real, liquid, and well matched to the regime, then discounts strikes that expose too much capital or create a weak recovery path.

Put composite // numerator: edge × liquidity × regime fit // denominator: product of penalty factors ≥ 1 score = (Edge · Liq · RegimeFit) ─────────────────────────────────── (Πimp · Πprox · Πfloor · Πrec) Edge = Extrinsic$ · VRPi / (Weeks · Capital) VRPi = clamp(IV/RVanchor − 1, 0, 1.50) Πimp = 1 + λimp(s) · Ii² Πprox = support × gamma × event threshold Πfloor= 1 + λfloor · (shortfall/EMH) Πrec = 1 + λrec · RBi
Call score · recovery phase

Plain language: the call side only counts decayable premium as edge. It does not confuse stock like carry above cost basis with option income.

Call composite — extrinsic only // intrinsic above cost basis is not "edge" score = (Extrinsic$ · VRP · Liq · Recovery) ────────────────────────────────────── (Weeks · Capital · Πtail) // Retrospective P&L accounting carry = max(0, (spot − costBasis) · PV) deadZone = spot < costBasis exitPnL = totalPutPrem + totalCallPrem + (strike − costBasis) · PV recommend = argmax{ score(c) : c passes survivability }
Catastrophic tail · QH

Plain language: this is the system’s disaster denominator. It estimates the move size WISE should still survive over the holding horizon, especially around gap risk and event risk.

Horizon tail move — the denominator of catastrophe defense H = clamp(round(DTE),  1,  5) Qrv = zα · σH · spot Qhist = CVaR99(gap / H-day) · spot Qevent = mevent · Qrv QH = max(Qrv, Qhist, Qevent) Ii = PV · max(QH − cushionBE, 0) / Capital SURVIVABILITY → reject when Ii > Imax(supportBand)
Capabilities

Four capabilities that support one recommendation.

Put phase

Phase aware put scorer

A composite scoring model where edge comes from per strike VRP and decayable premium, then gets discounted by impairment, proximity, floor, and recovery burden.

  • Support ratio aware proximity thresholds
  • Horizon conditioned tail move QH
  • Break even above spot gate calibrated by support and regime
  • Dynamic VRP calibration from VIX term, gamma, and IV rank
Call phase

Call recovery scorer

Extrinsic only logic so intrinsic value above cost basis never masquerades as edge. The engine still displays carry, dead zone status, and full cycle economics honestly.

  • Carry advantage computed from spot and cost basis
  • Dead zone flag when spot sits below cost basis
  • Full retrospective P&L with put premium, call premium, and underlying
  • Top candidate and skip reasons remain auditable
Before chain download

DTE advisor

Before any chain is downloaded, WISE ranks DTE 0 to 5 by fit to regime, events, gamma, and volatility so analysis starts in the right expiry.

  • CarryFit reflects theta decay quality by DTE
  • EventFit checks overlap with risk windows
  • GammaFit uses dealer positioning
  • VolFit reflects implied and realized volatility posture
Risk simulation

Trade fragility simulator

A path tool that tests the actual position instead of an abstract strategy. It is used to understand fragility, not to promise future profit.

  • VRP calibrated realized volatility for daily steps
  • Touch, breach, and assignment probabilities
  • Max adverse excursion and expiry CVaR
  • Configurable paths, strike override, and breach threshold
How WISE thinks

Three decisions WISE keeps separate. That is why the output stays honest.

A trade can be analytically attractive and still be blocked by risk controls. Or events can be clear while the chain offers no edge. WISE keeps those judgments separate so the user can see what is true, what is actionable, and what is changing.
